# Глоссарий: Операционные системы

---

**Операционная система (ОС)** — комплекс программ, обеспечивающий управление аппаратными ресурсами компьютера и предоставляющий интерфейс для взаимодействия пользователя и приложений с оборудованием.

**Ядро (Kernel)** — центральная часть операционной системы, которая постоянно находится в памяти и управляет процессами, памятью, устройствами и системными вызовами.

**Драйвер (Driver)** — программа, обеспечивающая взаимодействие операционной системы с конкретным устройством (принтером, видеокартой, звуковой картой).

**Процесс (Process)** — программа во время выполнения, включающая код, данные, стек и контекст выполнения.

**Поток (Thread)** — наименьшая единица выполнения внутри процесса; несколько потоков одного процесса разделяют память и ресурсы.

**Планировщик (Scheduler)** — компонент ОС, который определяет, какой процесс получит доступ к процессору и на какое время.

**Контекстное переключение (Context Switch)** — механизм переключения процессора с одного процесса на другой с сохранением состояния текущего процесса и восстановлением состояния следующего.

**Состояния процесса (Process States)** — режимы, в которых может находиться процесс: создание, готовность, исполнение, ожидание, завершение.

**Виртуальная память (Virtual Memory)** — абстракция, позволяющая программе использовать больше памяти, чем доступно физически, за счет использования дискового пространства.

**Страница (Page)** — фиксированный блок виртуальной памяти (обычно 4 КБ), которым управляет менеджер памяти.

**Файл подкачки (Page File / Swap)** — файл или раздел на диске, используемый для хранения страниц памяти, выгруженных из оперативной памяти.

**Страничное прерывание (Page Fault)** — ситуация, когда программа обращается к странице памяти, которая в данный момент находится не в RAM, а на диске.

**Менеджер памяти (Memory Manager)** — компонент ОС, отвечающий за распределение оперативной и виртуальной памяти между процессами.

**Файловая система (File System)** — способ организации, хранения и управления данными на носителе (диске, SSD, флешке).

**Файл (File)** — именованная область данных на носителе, содержащая информацию.

**Каталог (Directory)** — специальный файл, содержащий список других файлов и каталогов (папок).

**Inode** — структура данных в Unix-подобных ОС, содержащая метаданные файла (размер, права доступа, время создания, расположение на диске).

**Кластер (Cluster)** — минимальная единица дискового пространства, которую может выделить файловая система (обычно 4 КБ).

**Журналирование (Journaling)** — механизм файловой системы, при котором перед изменением данных записывается запись в журнал для восстановления после сбоев.

**CLI (Command Line Interface)** — интерфейс взаимодействия с ОС через ввод текстовых команд.

**GUI (Graphical User Interface)** — графический интерфейс взаимодействия с ОС, использующий окна, иконки, меню и указатель.

**Shell** — командная оболочка, интерпретатор команд, обеспечивающий взаимодействие пользователя с ОС через CLI.

**Многозадачность (Multitasking)** — способность ОС выполнять несколько процессов одновременно, создавая иллюзию параллельной работы.

**Системный вызов (System Call)** — интерфейс, через который прикладные программы запрашивают у ОС выполнение привилегированных операций (работа с файлами, памятью, устройствами).

**Права доступа (Permissions)** — набор правил, определяющих, какие пользователи и процессы могут читать, записывать или выполнять файл.
